# 行为准则

`CX-Codex` 是面向自托管 Codex Web / Android 使用场景的开源项目。我们希望 Issue、PR、讨论和 Release 协作保持专业、清晰、可验证。

## 我们鼓励

- 尊重不同经验背景的使用者和贡献者。
- 围绕可复现问题、日志、版本、截图和验证结果讨论。
- 在安全、远程访问、App Server 协议、凭据处理和移动端体验问题上保持谨慎。
- 给出具体建议、替代方案和回滚方式，而不是只做情绪化评价。
- 主动脱敏 Token、Cookie、公网地址、个人目录、业务文件和私人会话内容。

## 不接受

- 人身攻击、侮辱、骚扰、威胁或歧视性表达。
- 在公开 Issue、PR、截图、日志或 Release 说明中泄露他人或自己的敏感信息。
- 要求维护者复现无法公开验证的私人环境，同时拒绝提供脱敏后的必要信息。
- 诱导用户关闭密码、绕过 Codex sandbox / approval、安全提示或访问控制。
- 大量重复、偏离主题或明显破坏协作秩序的内容。

## 执行方式

维护者可以编辑或隐藏不合适内容、要求补充脱敏信息、锁定讨论、关闭 Issue / PR，或在严重情况下限制继续参与。处理时优先保护用户安全、凭据安全和项目协作效率。

## 报告方式

- 安全漏洞和凭据泄露：按 [SECURITY.md](./SECURITY.md) 的方式处理，不要公开完整利用细节。
- 普通协作问题：在相关 Issue / PR 中提醒维护者，并尽量附上上下文链接。
- 安装、同步、Android 或远程访问问题：先按 [SUPPORT.md](./SUPPORT.md) 收集脱敏诊断信息。

本准则适用于本仓库的 Issue、PR、讨论、Release 说明、文档协作和相关社区入口。
