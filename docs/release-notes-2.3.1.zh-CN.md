# CX-Codex 2.3.1：安卓语音输入稳定性更新

本版本集中修复手机浏览器和 Android App 中“点击麦克风没有反应”“系统语音服务失败后只能手动上传”等问题。

## 主要更新

- Android App 不再依赖不同厂商稳定性不一的系统 `SpeechRecognizer`。现在由 App 原生录音，点击“完成”后自动提交 CX-Codex 转写接口并将文字填入输入框。
- Android 连接普通 HTTP 自托管服务时也能使用语音输入，不受 WebView 安全上下文限制，正常流程无需手动选择或上传音频。
- 修复移动浏览器触摸事件阻止麦克风按钮点击，以及 HTTP 音频选择器延迟触发导致用户手势授权丢失的问题。
- 新增紧凑语音状态操作条，明确展示等待权限、录音、转写、成功和失败状态，并提供“完成”“取消”“选择音频”等恢复操作。
- 转写文字默认保留在输入框供编辑确认，不会自动发送。

## 使用方式

1. 安装本版本 Android APK。
2. 首次使用时允许麦克风权限。
3. 点击输入框右侧麦克风开始说话，再点击“完成”。
4. 等待文字自动进入输入框，确认或编辑后发送。

Android 原生能力无法通过网页热更新，因此旧 APK 用户需要安装本版本 APK 才能获得新的自动录音转写链路。
